Autenticación

Aprende cómo autenticar tus peticiones a la API de TuPay utilizando API Keys de forma segura y eficiente.

Métodos de Autenticación

TuPay API utiliza API Keys para autenticar todas las peticiones. Tu API Key identifica tu cuenta y te da acceso a los recursos de la API.

¡Mantén tu API Key segura!

Tu API Key tiene acceso completo a tu cuenta. Nunca la compartas públicamente en repositorios, código del lado del cliente, o cualquier lugar donde pueda ser expuesta.

Mejores Prácticas de Seguridad

Hacer ✓

  • Usa variables de entorno para almacenar tu API Key
  • Rota tus API Keys periódicamente
  • Usa diferentes keys para desarrollo y producción
  • Implementa rate limiting en tu aplicación
  • Monitorea el uso de tu API Key en el dashboard

No Hacer ✗

  • Nunca hagas commit de API Keys en Git
  • No incluyas API Keys en código del frontend
  • No compartas tu API Key por email o chat
  • No uses la misma key en múltiples proyectos
  • No dejes API Keys en archivos de log

Errores de Autenticación

Si tu API Key es inválida o falta, recibirás un error 401 Unauthorized:

Causas Comunes

  • API Key incorrecta o mal copiada
  • Header x-api-key faltante
  • API Key deshabilitada o revocada
  • Usando key de pruebas en producción (o viceversa)

Configuración de Webhooks

Los webhooks te permiten recibir notificaciones en tiempo real cuando ocurren eventos en tus transacciones. Configúralos directamente desde el dashboard.

1

Accede a Integraciones

En el dashboard, haz clic en "Integraciones" en el menú de tu perfil.

2

Configura tus URLs

Encontrarás dos campos para configurar webhooks:

URL de Webhook para PayIn

Recibe notificaciones de pagos entrantes (depósitos)

https://tu-servidor.com/webhooks/payin

URL de Webhook para PayOut

Recibe notificaciones de pagos salientes (retiros)

https://tu-servidor.com/webhooks/payout
3

Guarda la configuración

Haz clic en guardar y tu servidor comenzará a recibir notificaciones automáticamente cuando cambien los estados de tus transacciones.

Requisitos del Webhook

Tu servidor debe responder con un código HTTP 200 para confirmar la recepción. Si falla, TuPay reintentará el envío varias veces.

Webhook de PayIn (Pago Entrante)

Cada vez que se crea o actualiza una transacción de pago entrante (PayIn), TuPay enviará una notificación a la URL que hayas configurado con la siguiente estructura:

Descripción de Campos

Campo Tipo Descripción
type string Siempre "pay-in" para pagos entrantes
transactionId string Identificador único de la transacción
userId string Identificador del usuario asociado
amount number Monto solicitado de la transacción
status string Estado actual de la transacción
timestamp string Fecha y hora en formato ISO 8601
customId string Identificador personalizado que enviaste al crear la transacción
amountReceived number Monto efectivamente recibido
notes string Información adicional (puede estar vacío)

Webhook de PayOut (Pago Saliente)

Cuando se crea o actualiza una transacción de pago saliente (PayOut), recibirás una notificación con la siguiente estructura:

Descripción de Campos

Campo Tipo Descripción
type string Siempre "pay-out" para pagos salientes
transactionId string Identificador único de la transacción
userId string Identificador del usuario asociado
reference string Referencia de la transacción
amount number Monto de la transacción
status string Estado actual de la transacción
timestamp string Fecha y hora en formato ISO 8601
customId string Identificador personalizado que enviaste al crear la transacción
observation string Observaciones adicionales (puede estar vacío)

Webhook de PayOut Masivo

Cuando se procesan pagos salientes de forma masiva, recibirás una única notificación que contiene el resumen de todas las transacciones del lote:

Descripción de Campos

Campo Tipo Descripción
type string Siempre "pay-outs" para lotes masivos
totalTransactions number Cantidad total de transacciones en el lote
transactions array Lista de transacciones individuales con su estado

Nota sobre PayOut Masivo

Cada transacción dentro del arreglo transactions contiene los campos transactionId, reference, amount y status. Revisa el estado individual de cada transacción ya que pueden variar dentro del mismo lote.

Estados de Transacción - PayIn

Las transacciones de pago entrante (PayIn) pasan por los siguientes estados. Recibirás un webhook cada vez que el estado cambie.

Estado Descripción
pending La transacción fue creada y está esperando el pago del cliente
approved El pago fue detectado y está siendo verificado
paid El pago fue confirmado exitosamente
rejected La transacción fue rechazada
cancelled La transacción fue cancelada
expired La transacción expiró por falta de pago

Flujo Normal

pending → approved → paid

Estado final exitoso

El estado paid indica que el pago fue completado correctamente. Usa este estado para confirmar la operación en tu sistema.

Estados de Transacción - PayOut

Las transacciones de pago saliente (PayOut) pasan por los siguientes estados. Recibirás un webhook cada vez que el estado cambie.

Estado Descripción
pending La transacción fue creada y está en espera de aprobación
approved La transacción fue aprobada y será procesada
processing La transacción está siendo procesada
paid El pago fue enviado al destinatario
completed La transacción fue completada exitosamente
rejected La transacción fue rechazada
failed La transacción falló durante el procesamiento
cancelled La transacción fue cancelada

Flujo Normal

pending → approved → processing → paid → completed

Estado final exitoso

El estado completed indica que el pago saliente fue completado correctamente. Usa este estado para confirmar la operación en tu sistema.

Verificación de Firma (Seguridad)

Cada webhook enviado por TuPay incluye una firma digital HMAC-SHA256 para que puedas verificar que la notificación es auténtica. Es altamente recomendable que valides esta firma en cada webhook que recibas.

Headers de Seguridad

Header Descripción
X-TuPay-Signature Firma HMAC-SHA256 del contenido del webhook
X-TuPay-Timestamp Marca de tiempo del momento en que se envió el webhook

Importante

La firma se genera concatenando el timestamp y el cuerpo del webhook separados por un punto: timestamp.payload. Utiliza tu Secret Key como clave para el HMAC.

Recomendaciones

Buenas Prácticas

  • Responde siempre con HTTP 200 lo antes posible
  • Valida la firma HMAC-SHA256 en cada webhook recibido
  • Implementa idempotencia usando el transactionId
  • Procesa los webhooks de forma asíncrona
  • Registra todos los webhooks recibidos para auditoría

Evitar

  • No realices procesos largos antes de responder al webhook
  • No ignores la verificación de firma
  • No asumas que los webhooks llegarán en orden
  • No proceses el mismo webhook más de una vez sin idempotencia
  • No expongas tu Secret Key en código del cliente

Reintentos Automáticos

Si tu servidor no responde con un código 200, TuPay reintentará el envío del webhook automáticamente. Asegúrate de que tu lógica soporte recibir el mismo webhook más de una vez (idempotencia).
